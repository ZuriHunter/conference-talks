What is Docker?

Docker is a platform that helps developers build, ship and run any application, in any environment. Docker containers wrap a piece of software in a complete filesystem that contains everything needed to run:

  • Code
  • Runtime
  • System Tools
  • System Libraries
  • Docker Inc, Sponsor of Docker Platform
  • Offered as a Containers as a Service (CaaS)
  • 7 billion Docker Image Downloads
  • 3,000 Contributors
  • 700,000 Applications created using Docker

It is not a Virtual Machine

Virtual Machines (VM) are an emulation of a computer system. Like computers it has and runs its own operating system and applications to operate.

  • Both provide isolated environment for applications to run inside.
  • Both are easily movable between different hosts.

The Trend

Monolithic

  • Cost a lot to run
  • Hard code base to debug and maintain
  • Unnecessary shared scope of responsibilites
  • Inflexible

Microservice

  • Smaller and Easier to run
  • Each service has a clearly defined set of responsibilities
  • Easy code base to maintain
  • 1 Scope = 1 Microservice

    • Dilemmas with Microservice

    • Each service may have a different language or tech stack
    • Starting an application can be difficult due to the number of services needed to support the application
    • Each service may have its own dependencies and requirements

    Docker's Role

    Build,Run,Ship---Anywhere

    • Makes it easy for developers to write code that they can run anywhere by relyong on software containers
    • Containers are light weight and portable
    • Containers share the resources of the computer running them

    Run Your 2nd Container

    Docker Image
    Is a file system and configuration of our application which are used to create containers.

    Alpine is a lightweight Linux distribution.

    docker pull alpine

    Lets run a docker container base off of this image

    docker run alpine ls -l

    What Happened?

    1. Docker daemon pulls the "Alpine" image from Docker Hub.
    2. Generates a container base off the Alpine image.
    3. Executes the command within the container.

    Try this:

    docker run alpine [COMMAND]

    echo "Hello WWC"
    pwd
    id -u -n
    cal

    To interact within the docker container:

    docker run -it alpine /bin/sh

    • cd usr

    Docker + Flask

    1. Create a docker image of the application.
    2. Build the docker image of the application.
    3. Run a container base off the image of the application.

    Open the terminal or command prompt

    git clone https://github.com/ZuriHunter/docker-flask.git

    Open the application in the text editor.

    At the root of the application create a file called "Dockerfile".

    Dockerfile is a text file that contains a list of commands that Dockers calls while creating the image. This file contains all the information that Docker needs to know to run the app.

    Dockerfile Commands

    • ADD
    • COPY
    • FROM
    • CMD
    • LABEL
    • EXPOSE
    • ENV
    • ENTRYPOINT
    • VOLUME
    • USER
    • ARG
    • WORKDIR
    • ONBUILD
    • ARG
    • STOPSIGNAL
    • SHELL
    • HEALTHCHECK

    Add this to the Dockerfile:

    FROM alpine:3.5

    RUN apk add --update py2-pip
    COPY requirements.txt /usr/src/app/

    RUN pip install --no-cache-dir -r /usr/src/app/requirements.txt
    COPY app.py /usr/src/app/

    COPY templates/index.html /usr/src/app/templates/
    EXPOSE 5000

    CMD ["python", "/usr/src/app/app.py"]
    docker build -t dockerflask .

    What Happened?

    1. Docker finds the Dockerfile at the root of the application to build the image.
    2. Docker builds the image and tags it with the name "dockerflask".
    docker run -p 8888:5000 dockerflask

    What Happened?

    1. Docker runs a container base off the image "dockerflask".
    2. Docker then forwards port 8888 to port 5000 that was exposed on the image.
    http://localhost:8888
